# US-012: Refresh Token

#### Jako użytkownik chcę korzystać z refresh tokenów, aby nie musieć logować się co kilkanaście minut/kilka godzin. Zadanie: TELE1-89

**Kryteria akceptacji:**
- [ ] Przy logowaniu tworzy się również refresh token
- [ ] Refresh token jest trzymany w bazie danych
- [ ] JWT odnawia się automatycznie w przypadku odświeżenia autoryzacji przez refresh token
- [ ] Po wylogowaniu Refresh token usuwa się z bazy danych
- [ ] Przy odnowieniu tokena, stary się usuwa i tworzy nowy
