# US-003: Zamiana pola source na typ predefiniowany

#### Jako programista/system, chcę zastąpić obecne wolnotekstowe pole source w encjach i DTO stron telegazety typem predefiniowanym (enumem lub encją), aby pole było walidowane i ograniczone do dozwolonych wartości — dzięki temu zminimalizujemy ryzyko zapisania błędnych danych do bazy. Zadanie: TELE1-78

**Kryteria akceptacji:**
- [ ] Znalezione i udokumentowane wszystkie miejsca użycia pola source (encje, DTO, mappingi, serwisy, repozytoria, migracje).
      
- [ ] Wprowadzony typ predefiniowany (enum lub encja) reprezentujący dozwolone źródła.

- [ ] Utworzona migracja bazy danych konwertująca/ograniczająca istniejące dane do nowego formatu lub dodająca kolumnę/zależność, bez utraty poprawnych danych.

- [ ] Wszystkie encje/DTO oraz warstwy API zostały zaktualizowane do używania nowego typu zamiast wolnego stringa.

- [ ] Walidacja po stronie wejścia (request DTO / kontrolery) i po stronie encji wymusza tylko dozwolone wartości.
