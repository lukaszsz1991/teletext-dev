# Przebieg prac w podziale na Sprinty

## Sprint 1
* **Cel:** Konfiguracja środowiska i inicjalizacja szkieletu systemu.
* **Zakres prac:**
    * Konfiguracja projektu Spring Boot (TELE1-1)
    * Przygotowanie pliku Docker Compose (TELE1-2)
    * Inicjalizacja bazy danych PostgreSQL i Flyway (TELE1-3)
    * Stworzenie encji User oraz Role (TELE1-4)
    * Konfiguracja modułu Security - podstawy (TELE1-5)
    * Dokumentacja techniczna - struktura (TELE1-6)
    * Konfiguracja serwera Redis (TELE1-7)

## Sprint 2
* **Cel:** Budowa rdzenia silnika telegazety i zarządzania danymi.
* **Zakres prac:**
    * Struktura tabeli stron telegazety (TELE1-11)
    * Logika numeracji stron 100-899 (TELE1-12)
    * CRUD stron w panelu administratora (TELE1-13)
    * Implementacja wyszukiwania po kategorii (TELE1-14)
    * System autoryzacji login/password (TELE1-15)
    * Obsługa wyjątków i błędów systemowych (TELE1-16)

## Sprint 3
* **Cel:** Prace nad warstwą wizualną i pierwszą integracją API.
* **Zakres prac:**
    * Budowa layoutu CSS retro (TELE1-21)
    * Obsługa znaków specjalnych i ASCII Art (TELE1-22)
    * Integracja API Pogoda - pobieranie danych (TELE1-23)
    * Mapowanie danych zewnętrznych na tekst (TELE1-24)
    * Widok listy kategorii dla użytkownika (TELE1-25)
    * Walidacja formularzy w panelu admina (TELE1-26)

## Sprint 4 (9 zgłoszeń)
* **Cel:** Ukończenie tworzenia aplikacji, początek wdrożenia aplikacji na produkcję.
* **Zakres prac:**
    * Widok użytkownika - wyświetlania strony telegazety (TELE1-9)
    * Nawigacja użytkownika między stronami telegazety (TELE1-10)
    * Zarządzanie szablonami stron telegazety (TELE1-76)
    * Testy integracji – zewnętrzna API (Lotto, NBP, Pogoda, Horoskop...) (TELE1-79)
    * Testy statystyk stron (TELE1-85)
    * Zarządzanie stronami telegazety (TELE1-82)
    * Cachowanie danych (TELE1-87)
    * Refresh Token (TELE1-89)
    * Wysyłka mail (TELE1-80)
